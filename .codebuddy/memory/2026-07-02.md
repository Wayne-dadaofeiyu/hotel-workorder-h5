## 修复 0314 订单丢失（从第一性原理重构注入逻辑）(2026-07-02, ≈14:55)
### 问题诊断
- `sessionStorage` 标记在页面刷新/重新加载后保留，但 `state.orders` 被重新初始化（generateMockOrders 重跑）
- 两者不同步导致 0314 标记存在但订单不存在 → 切换 Cleaning tab 不创建 → "丢失"

### 修复方案
- 移除 `DEMO_0314_FIRED_KEY` 的 sessionStorage 机制
- 每次切换 Cleaning tab，检查 `state.orders` 中是否存在**任何** 0314 cleaning（不限定 status）
  - 不存在 → 创建（说明是新 session 或未创建）
  - 存在（pending 或 completed）→ 不动
- 页面刷新/重新进入 → useRef + state.orders 都归零 → 自动重新创建
- 同 session 内完成 0314 → orders 中存在 completed → 不再创建，仅在历史列表
